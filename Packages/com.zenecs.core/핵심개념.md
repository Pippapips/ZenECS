## 개요

**ZenECS Core**는 순수 C# 기반의 **Entity-Component-System (ECS)** 런타임입니다. Unity에 의존하지 않으며, 게임 엔진과 독립적인 .NET 애플리케이션 모두에서 사용할 수 있습니다.

### 주요 특징

- **멀티 월드 커널**: 여러 월드를 생성하고 관리하며, 결정론적(Deterministic) 실행
- **통합 월드 API**: 컴포넌트, 쿼리, 명령 버퍼, 메시지, 훅, 컨텍스트/바인더를 통합한 단일 API
- **의존성 없음**: 외부 프레임워크 없이 경량 내부 DI 및 워커 큐 사용
- **결정론적 스텝**: `BeginFrame → FixedStep×N → LateFrame` 구조와 명시적 순서 지정
- **스레드 안전성**: 동시 컬렉션과 락 기반 스냅샷을 사용한 안전한 멀티스레드 접근

## Kernel (커널)

**Kernel**은 여러 월드를 관리하고 프레임 틱을 조율하는 최상위 관리자입니다.

### 주요 역할

- 여러 월드 생성/삭제 및 ID/이름/태그로 조회
- 현재 월드 선택 및 변경 이벤트
- 일시정지/재개 지원
- 스레드 안전한 월드 인덱싱

### 기본 사용법

```csharp
using ZenECS.Core;

// 커널 생성
var kernel = new Kernel(new KernelOptions
{
    AutoSelectNewWorld = true,
    StepOnlyCurrentWhenSelected = false
});

// 월드 생성
var world1 = kernel.CreateWorld("World1");
var world2 = kernel.CreateWorld("World2");

// 현재 월드 설정
kernel.SetCurrentWorld(world1);

// 게임 루프
kernel.BeginFrame(dt: 1f / 60f);
kernel.FixedStep(fixedDelta: 1f / 60f);
kernel.LateFrame(alpha: 1f);

// 또는 간편한 방법
kernel.PumpAndLateFrame(dt, fixedDelta, maxSubStepsPerFrame);

// 정리
kernel.Dispose();
```

### 프레임 구조

```
BeginFrame (변수 타임스텝)
  ↓
FixedStep × N (고정 타임스텝, 시뮬레이션)
  ↓
LateFrame (프레젠테이션, 읽기 전용)
```

## World (월드)

**World**는 모든 ECS 기능을 통합한 단일 공개 API입니다. 하나의 시뮬레이션 공간을 나타냅니다.

### 주요 기능

- **Entities**: 엔티티 생성/삭제, 생명주기 관리
- **Components**: 컴포넌트 추가/제거/쿼리, 타입별 풀링
- **Query**: 빠른 엔티티 쿼리 및 필터링 지원
- **CommandBuffer**: 구조적 변경을 버퍼링하고 안전한 경계에서 적용
- **Messages**: 구조체 기반 Pub/Sub 메시징
- **Contexts & Binders**: 뷰 통합을 위한 바인딩 시스템
- **Hooks**: 쓰기 권한 및 검증 훅
- **Snapshot**: 저장/로드 및 마이그레이션
- **Reset**: 빠른 월드 리셋

### 기본 사용법

```csharp
// 월드 생성 (커널을 통해)
var world = kernel.CreateWorld("GameWorld");

// 엔티티 생성
var entity = world.CreateEntity();

// 컴포넌트 추가
world.AddComponent(entity, new Position { X = 0, Y = 0 });
world.AddComponent(entity, new Velocity { X = 1, Y = 0 });

// 컴포넌트 읽기/쓰기
ref var pos = ref world.Ref<Position>(entity);  // 참조로 수정
var vel = world.Get<Velocity>(entity);          // 값으로 읽기

// 컴포넌트 확인
bool hasPos = world.HasComponent<Position>(entity);

// 엔티티 삭제
world.DestroyEntity(entity);
```

## Entity (엔티티)

**Entity**는 컴포넌트를 담는 컨테이너입니다. 단순한 ID로 표현되며, 자체 데이터는 포함하지 않습니다.

### 특징

- 엔티티는 고유한 ID를 가집니다
- 엔티티 자체는 데이터를 포함하지 않습니다
- 컴포넌트를 통해 데이터를 저장합니다
- 엔티티는 생성/삭제될 수 있으며, 삭제된 엔티티는 재사용되지 않습니다

### 사용 예시

```csharp
// 엔티티 생성
var player = world.CreateEntity();
var enemy = world.CreateEntity();

// 엔티티 생존 확인
if (world.IsAlive(player))
{
    // 엔티티가 살아있음
}

// 엔티티 ID 접근
int id = player.Id;
```

## Component (컴포넌트)

**Component**는 데이터를 저장하는 구조체입니다. 값 타입이며 타입별로 분리된 풀에 저장됩니다.

### 특징

- 컴포넌트는 `struct` 값 타입이어야 합니다
- 불변성(immutability) 패턴을 권장합니다 (`readonly struct`)
- 타입별로 분리된 풀에 저장되어 메모리 효율적입니다
- 컴포넌트는 엔티티에 추가/제거할 수 있습니다

### 컴포넌트 정의 예시

```csharp
// 위치 컴포넌트
public readonly struct Position
{
    public readonly float X, Y;
    public Position(float x, float y)
    {
        X = x;
        Y = y;
    }
}

// 속도 컴포넌트
public readonly struct Velocity
{
    public readonly float X, Y;
    public Velocity(float x, float y)
    {
        X = x;
        Y = y;
    }
}

// 태그 컴포넌트 (데이터 없음)
public readonly struct Paused { }
```

### 컴포넌트 조작

```csharp
// 추가
world.AddComponent(entity, new Position(0, 0));

// 읽기 (값)
var pos = world.Get<Position>(entity);

// 읽기 (읽기 전용)
var pos = world.ReadComponent<Position>(entity);

// 수정 (참조)
ref var pos = ref world.Ref<Position>(entity);
pos = new Position(pos.X + 1, pos.Y);

// 설정
world.SetComponent(entity, new Position(10, 20));

// 제거
world.RemoveComponent<Velocity>(entity);

// 확인
bool hasPos = world.HasComponent<Position>(entity);
```

## System (시스템)

**System**은 게임 로직을 캡슐화하는 클래스입니다. 컴포넌트를 조회하고 수정하여 게임 상태를 업데이트합니다.

### 시스템 타입

ZenECS는 다음 시스템 인터페이스를 제공합니다:

1. **`IFrameSetupSystem`**: 프레임당 한 번 초기화
2. **`IFixedSetupSystem`**: 고정 스텝당 한 번 초기화
3. **`IVariableRunSystem`**: 변수 타임스텝 실행 (BeginFrame)
4. **`IFixedRunSystem`**: 고정 타임스텝 실행 (SimulationGroup)
5. **`IPresentationSystem`**: 프레젠테이션 단계 (LateFrame)
6. **`ISystemLifecycle`**: 생명주기 관리 (Initialize/Shutdown)

### 시스템 그룹

시스템은 그룹으로 분류됩니다:

- **`[SimulationGroup]`**: 시뮬레이션 단계 (FixedStep)
- **`[PresentationGroup]`**: 프레젠테이션 단계 (LateFrame)

### 시스템 순서 지정

```csharp
[SimulationGroup]
[OrderAfter(typeof(PhysicsSystem))]  // PhysicsSystem 이후 실행
[OrderBefore(typeof(RenderSystem))]   // RenderSystem 이전 실행
public sealed class MoveSystem : IFixedRunSystem
{
    public void Run(IWorld world, float fixedDelta)
    {
        // 이동 로직
    }
}
```

### 시스템 작성 예시

```csharp
using ZenECS.Core;
using ZenECS.Core.Systems;

[SimulationGroup]
public sealed class MoveSystem : IFixedRunSystem
{
    public void Run(IWorld world, float fixedDelta)
    {
        // Position과 Velocity를 가진 모든 엔티티 조회
        foreach (var entity in world.Query<Position, Velocity>())
        {
            ref var pos = ref world.Ref<Position>(entity);
            var vel = world.Get<Velocity>(entity);
            
            // 위치 업데이트
            pos = new Position(
                pos.X + vel.X * fixedDelta,
                pos.Y + vel.Y * fixedDelta
            );
        }
    }
}

[PresentationGroup]
public sealed class PrintPositionsSystem : IPresentationSystem
{
    public void Run(IWorld world, float dt, float alpha)
    {
        // 읽기 전용 조회
        foreach (var entity in world.Query<Position>())
        {
            var pos = world.ReadComponent<Position>(entity);
            Console.WriteLine($"Entity {entity.Id}: {pos}");
        }
    }
}
```

### 시스템 등록

```csharp
world.AddSystems([
    new MoveSystem(),
    new PrintPositionsSystem()
]);
```

## Query (쿼리)

**Query**는 특정 컴포넌트를 가진 엔티티를 효율적으로 조회하는 방법입니다.

### 기본 쿼리

```csharp
// 단일 컴포넌트 조회
foreach (var entity in world.Query<Position>())
{
    var pos = world.Get<Position>(entity);
}

// 여러 컴포넌트 조회
foreach (var entity in world.Query<Position, Velocity>())
{
    ref var pos = ref world.Ref<Position>(entity);
    var vel = world.Get<Velocity>(entity);
}
```

### 필터링

```csharp
using ZenECS.Core.Filters;

// 필터 생성
var filter = Filter.New
    .With<Position>()      // Position 컴포넌트 필요
    .With<Velocity>()      // Velocity 컴포넌트 필요
    .Without<Paused>()     // Paused 컴포넌트 없어야 함
    .Build();

// 필터 사용
foreach (var entity in world.Query<Position, Velocity>(filter))
{
    // 필터 조건을 만족하는 엔티티만 처리
}
```

### 성능 특징

- **제로 할당 쿼리**: 열거 가능한 쿼리는 힙 메모리를 할당하지 않습니다
- **최적화된 인덱싱**: 가장 작은 컴포넌트 풀을 기준으로 쿼리가 시드됩니다
- **비트셋 기반**: 컴포넌트 존재 추적을 위한 밀집 비트셋 사용

## Command Buffer (명령 버퍼)

**Command Buffer**는 구조적 변경(엔티티 생성/삭제, 컴포넌트 추가/제거)을 버퍼링하고 안전한 경계에서 일괄 적용하는 메커니즘입니다.

### 사용 시나리오

- 시스템 실행 중 구조적 변경을 안전하게 처리
- 멀티스레드 환경에서 변경 사항을 버퍼링
- 결정론적 실행을 위한 변경 지연

### 기본 사용법

```csharp
// 명령 버퍼 시작
using (var cmd = world.BeginWrite())
{
    // 구조적 변경 버퍼링
    cmd.AddComponent(entity, new Health { Value = 100 });
    cmd.RemoveComponent<Stunned>(entity);
    cmd.DestroyEntity(entity);
    
    // ReplaceComponent도 지원
    cmd.ReplaceComponent(entity, new Health { Value = 75 });
} // 자동으로 적용됨

// 명시적 적용
world.RunScheduledJobs();
```

### 즉시 적용

```csharp
using (var cmd = world.BeginWrite())
{
    cmd.AddComponent(entity, new Health { Value = 100 });
}
// EndWrite가 자동 호출되어 즉시 적용됨
```

## Message Bus (메시지 버스)

**Message Bus**는 구조체 기반 Pub/Sub 메시징 시스템입니다. 뷰 레이어에서 데이터 레이어로의 단방향 데이터 흐름을 구현합니다.

### 메시지 정의

```csharp
public readonly struct Damage : IMessage
{
    public readonly Entity Target;
    public readonly int Amount;
    
    public Damage(Entity target, int amount)
    {
        Target = target;
        Amount = amount;
    }
}
```

### 구독 및 발행

```csharp
// 구독
var subscription = world.Subscribe<Damage>(damage =>
{
    // 데미지 처리
    if (world.IsAlive(damage.Target) && world.HasComponent<Health>(damage.Target))
    {
        var health = world.Get<Health>(damage.Target);
        var newHealth = new Health(health.Value - damage.Amount);
        world.SetComponent(damage.Target, newHealth);
    }
});

// 발행
world.Publish(new Damage { Target = entity, Amount = 10 });

// 구독 해제
subscription.Dispose();
```

### 사용 패턴

메시지 버스는 **View → Data** 단방향 흐름을 구현합니다:

- **View 레이어**: World를 직접 수정하지 않고 메시지만 발행
- **Simulation 레이어**: 메시지를 구독하고 컴포넌트를 업데이트
- **Presentation 레이어**: 읽기 전용으로 데이터를 표시

## Binding (바인딩)

**Binding**은 뷰 통합을 위한 시스템입니다. Contexts와 Binders를 통해 ECS 데이터와 뷰(Unity, UI, 오디오 등)를 연결합니다.

### 주요 개념

- **Context**: 뷰 관련 데이터를 담는 컨테이너
- **Binder**: 컴포넌트 변경을 감지하고 뷰를 업데이트
- **ComponentDelta**: 컴포넌트 변경 사항 (이전 값, 새 값)

### 바인더 작성 예시

```csharp
using ZenECS.Core.Binding;

public sealed class SpriteBinder : BaseBinder, IBinds<Position>, IRequireContext<SpriteContext>
{
    public void OnDelta(in ComponentDelta<Position> delta)
    {
        // 컴포넌트 변경 시 뷰 업데이트
        var sprite = GetContext<SpriteContext>().Sprite;
        sprite.transform.position = new Vector3(
            delta.NewValue.X,
            delta.NewValue.Y,
            0
        );
    }
}
```

### 컨텍스트 정의

```csharp
public class SpriteContext : IContext
{
    public SpriteRenderer Sprite { get; set; }
}
```

## Hooks & Validators (훅과 검증기)

**Hooks**와 **Validators**는 월드 범위의 권한 체크와 값 검증을 제공합니다.

### Write Permission Hook

컴포넌트 쓰기 권한을 제어합니다:

```csharp
// 쓰기 권한 체크
world.Hooks.AddWritePermission((entity, componentType) =>
{
    // GodMode 컴포넌트는 쓰기 금지
    return componentType != typeof(GodMode);
});
```

### Validator

컴포넌트 값의 유효성을 검증합니다:

```csharp
// Health 값 검증
world.Hooks.AddValidator<Health>(health =>
{
    // Health는 0 이상이어야 함
    return health.Value >= 0;
});
```

### 사용 예시

```csharp
// 잘못된 값 설정 시도 (검증 실패)
world.SetComponent(entity, new Health { Value = -10 }); // 검증 실패로 예외 발생

// 권한 없는 컴포넌트 설정 시도
world.AddComponent(entity, new GodMode()); // 권한 체크 실패로 예외 발생
```

## Snapshot I/O (스냅샷 입출력)

**Snapshot I/O**는 월드 상태를 저장하고 로드하는 기능입니다. 플러그 가능한 백엔드와 포맷터를 지원합니다.

### 저장

```csharp
using (var stream = File.Create("save.dat"))
{
    world.Save(stream, new BinaryComponentFormatter());
}
```

### 로드

```csharp
using (var stream = File.OpenRead("save.dat"))
{
    var migrations = new List<IPostLoadMigration> 
    { 
        new V1ToV2Migration() 
    };
    world.Load(stream, new BinaryComponentFormatter(), migrations);
}
```

### Post-Load Migration

로드 후 데이터 마이그레이션을 수행할 수 있습니다:

```csharp
public class V1ToV2Migration : IPostLoadMigration
{
    public void Migrate(IWorld world)
    {
        // 이전 버전의 데이터를 새 버전으로 변환
        foreach (var entity in world.Query<OldComponent>())
        {
            var old = world.Get<OldComponent>(entity);
            world.RemoveComponent<OldComponent>(entity);
            world.AddComponent(entity, new NewComponent(old.Data));
        }
    }
}
```

## 빠른 시작 예제

전체 예제는 `Packages/com.zenecs.core/Samples~/01-Basic/Basic.cs`를 참고하세요.

```csharp
using ZenECS.Core;
using ZenECS.Core.Systems;

// 1. 커널 및 월드 생성
var kernel = new Kernel();
var world = kernel.CreateWorld("Game");

// 2. 컴포넌트 정의
public readonly struct Position { public float X, Y; }
public readonly struct Velocity { public float X, Y; }

// 3. 시스템 작성
[SimulationGroup]
public sealed class MoveSystem : IFixedRunSystem
{
    public void Run(IWorld world, float fixedDelta)
    {
        foreach (var entity in world.Query<Position, Velocity>())
        {
            ref var pos = ref world.Ref<Position>(entity);
            var vel = world.Get<Velocity>(entity);
            pos.X += vel.X * fixedDelta;
            pos.Y += vel.Y * fixedDelta;
        }
    }
}

// 4. 엔티티 및 컴포넌트 생성
var entity = world.CreateEntity();
world.AddComponent(entity, new Position { X = 0, Y = 0 });
world.AddComponent(entity, new Velocity { X = 1, Y = 0 });

// 5. 시스템 등록
world.AddSystems([new MoveSystem()]);

// 6. 게임 루프
const float fixedDelta = 1f / 60f;
while (running)
{
    float dt = GetDeltaTime();
    kernel.PumpAndLateFrame(dt, fixedDelta, maxSubStepsPerFrame: 4);
}

// 7. 정리
kernel.Dispose();
```
